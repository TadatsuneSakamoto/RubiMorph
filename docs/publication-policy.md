# Publication Policy

RubiMorph の公開候補には、利用者と開発参加者がプロジェクトを理解、実行、検証、再配布するために必要な情報だけを含めます。

## 公開してよい情報

- ソースコード、テスト、スキーマ、サンプルプロファイル、公開ドキュメント。
- 正式な作者名、プロジェクト名、公開Webサイト、公開GitHub URL。
- 利用者向けのビルド、テスト、配布、ライセンス確認手順。
- 配布物に必要なライセンス、通知、SBOM、チェックサム。

## 公開しない情報

- 認証情報、アクセストークン、パスワード、Cookie、秘密鍵、証明書秘密鍵。
- 開発者PC固有の絶対パス、個人ユーザー名を含むパス、ローカル管理パス。
- 内部作業記録、AI作業メモ、個人的な再開手順、未公開Issue相当のメモ。
- 非公開原稿、公開保留中のデータ、実在ユーザーの本文、個人情報。
- サーバー接続情報、リモートログインユーザー名、サーバーIP、内部デプロイ先、バックアップ先。
- build cache、crash dump、一時ファイル、IDE固有の個人設定。

## 記載ルール

- サンプルでは、実在するローカル絶対パスではなく、リポジトリ相対パスや `<repository-root>`、`<working-directory>`、`<web-root>` などのプレースホルダーを使います。
- ユーザー設定領域を説明する場合は `%APPDATA%` や `%LOCALAPPDATA%` などの環境変数表記を使います。
- Release Assets、Portable ZIP、インストーラー同梱ファイル、SBOM、チェックサムも公開前検査の対象にします。
- 公開前に `scripts/check_public_tree.py` と人手確認の両方を行います。
